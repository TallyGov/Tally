# Proposal Digest Format

Tally should make governance readable in one pass, especially for operators who need the consequences faster than the forum thread.

## Each digest should include

- The proposal title and current stage.
- The exact action being requested.
- The likely winners, losers, or risk shifts if it passes.
- The deadline or vote window that matters operationally.

## Keep out

- Long neutral summaries that hide the decision.
- Background history that does not change the vote outcome.
- Repeated quotes from the source thread when one citation is enough.

## Goal

The digest should let someone understand what is changing and whether they need to react before they open the full proposal discussion.
